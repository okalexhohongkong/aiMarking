## 本机数据位置

Mac App 运行后，数据放在：

```text
/Users/apple/Library/Application Support/wecom-ai-customer-service/data
```

平台凭证配置中心保存的本机配置放在：

```text
/Users/apple/Library/Application Support/wecom-ai-customer-service/.env
```

这样安装后的 App 不依赖源码目录，也不会因为系统权限导致数据写入失败。配置页面只显示已配置/未配置，不回显密钥明文；保存后建议重启 App，让真实接入服务重新读取配置。

## Mac 打包命令

```bash
npm run desktop:pack:mac
npm run desktop:dist:mac:arm64
npm run desktop:dist:mac:x64
```

`desktop:pack:mac` 生成可双击运行的 `.app`。

`desktop:dist:mac:arm64` 生成 Apple 芯片 Mac 的 `.dmg`。

`desktop:dist:mac:x64` 生成 Intel Mac 的 `.dmg`。

## Windows 和 Linux

Windows 和 Linux 已经预留 Electron 打包脚本：

```bash
npm run desktop:pack:win:x64
npm run desktop:pack:win:arm64
npm run desktop:pack:linux:x64
```

当前已经在 Mac 上交叉生成安装包。最稳妥的方式仍然是在对应系统上各自实际打开验证：

- Windows：在 Windows 电脑或虚拟机生成 `.exe`/安装包，并实际打开验证。
- Linux：在 Linux 环境生成 AppImage，并实际打开验证。

## iPad、安卓 Pad、小程序

这三类不能直接由当前桌面包一键转换：

- iPad：当前先用 PWA 轻量版，后续可做 Capacitor iOS 壳并进入 TestFlight。
- 安卓 Pad：当前先用 PWA 轻量版，后续可做 Capacitor Android 壳并生成 APK。
- 小程序：需要按微信小程序/抖音小程序规则重写页面和接口适配。
