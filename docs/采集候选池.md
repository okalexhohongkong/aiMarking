# 采集候选池

最近更新时间：2026-06-22

这个文档只记录采集板块的参考想法，不代表已经进入 1.0 开发范围。任何采集来源、采集规则、内容吸收策略，正式加入系统前都必须先和用户确认。

## 执行规则

1. 用户提供的新平台、新资料、新采集思路，先放入候选池。
2. 候选池内容不自动进入开发，不影响当前企业微信和抖音接入主线。
3. 如果我判断某个采集来源值得吸收，必须先用中文说明原因、影响范围和预计耗时。
4. 只有用户明确确认后，才允许把候选内容加入正式需求或开发计划。
5. 未确认前，不写入 1.0 主线、不改代码、不改接口。

| 层级 | 用户补充内容 | 候选理解 | 当前处理 |
| --- | --- | --- | --- |
| 数据采集层 | 评论爬虫、竞品数据 | 可能用于汇总自有评论、竞品评论和行业评论线索 | 候选，不开发 |

| 监控系统 | OpenClaw 告警、邮件推送 | 可能用于异常告警、日报周报和进度汇报 | 候选，不默认采用 |

候选链路：

```text
数据采集层 -> 数据解析层 -> 触达层 -> 监控系统
评论爬虫/竞品数据 -> 账号信息提取/画像分析 -> 自动私信/定时任务 -> OpenClaw告警/邮件推送
```

注意：这条链路目前只是候选方案，不进入当前企业微信和抖音客服接入主线。尤其是评论爬虫、自动私信、定时任务这些动作，后续如需采用，必须先确认平台规则、账号安全和人工审批机制。
