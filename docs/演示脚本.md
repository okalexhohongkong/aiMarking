# 黑卫士七维AI营销系统 Demo 演示流程

这份脚本用于把当前 1.0 演示成“可安装、可打开、可讲清楚”的版本。演示时不需要真实企业微信、抖音、淘宝等平台凭证；先展示本机后台、知识库、平台凭证配置中心、端口模拟、自动私信生成器、私信审批队列、知识库导入模板、客户生命周期、私信/评论转化剧本、Hermes 指令收件箱 / 双向中心、并行任务调度和营销闭环。

## 1. 演示前检查

```bash
npm install
npm run demo:check
npm test
npm run admin
```

打开：

```text
http://127.0.0.1:8787
```

如果 `8787` 端口被占用，先关闭旧的管理台，再重新执行 `npm run admin`。

## 2. 开场说明

推荐这样介绍：

```text
这是黑卫士七维AI营销系统的 1.0 Demo。当前先把知识库、知识图谱、多平台客服端口、自动私信生成器、客户生命周期、私信/评论转化剧本、Hermes 指令收件箱 / 双向中心、并行任务调度和后台 UI 打通。真实平台 API、批量发布、私信自动化暂时不直接发送，默认人工确认，先保证能演示、能配置、能逐步接入。
```

## 3. 点击顺序

1. 运行状态
   - 看机器人名称、知识库名称、接口地址、知识条目数和消息记录数。
   - 说明：Demo 可不填企业微信凭证，只演示后台和模拟端口。
   - 在“本机工具”里点“模板”，生成知识库导入模板；点“备份”和“导出”，说明本机数据可保护、可迁移。

2. 界面设置
   - 调整主色、背景、字体和字号。
   - 点“恢复默认”回到绿色主题。
   - 说明：后台可以按演示场景或品牌风格快速换视觉。

3. 知识库
   - 展示标题、分类、标签、适用场景、关键概念、处理步骤、内容。
   - 新增一条简单知识，例如：

```text
标题：Demo 接待说明
分类：演示
标签：demo, 接待
适用场景：演示客户提问
关键概念：知识库, 结构化回答
处理步骤：
先识别客户问题
再调用相关知识
最后给出下一步建议
内容：Demo 阶段系统会先根据知识库回答，并在不确定时提示人工确认。
```

4. 问答测试
   - 输入：`合同怎么签？`
   - 看回答是否包含结论、处理步骤、补充说明、参考来源。
   - 说明：没有外部大模型时，系统也会用本地知识兜底。

5. 知识图谱
   - 看知识与分类、场景、概念、步骤之间的关系。
   - 说明：后续行业知识库会靠这些字段形成体系化回答。

6. 多平台客服端口
   - 先看“平台凭证配置中心”：企业微信、抖音、个人微信、小红书、快手、视频号、电商、短信、资讯平台和海外社媒都按平台分组展示。
   - 说明：这里可以填写真实平台凭证；页面只显示已配置/未配置，不回显密钥明文。保存后重启后台或桌面 App，再进行真实平台联调。
   - 展示企业微信、小程序、个人微信、抖音、快手、小红书、视频号、淘宝、拼多多、京东、短信、头条号、百家号、B站、知乎、LinkedIn、Facebook、WhatsApp、X/Twitter、TikTok。
   - 说明：现在先预留端口和配置状态，真实平台 API 后续逐个接入。

7. 端口模拟器
   - 选择“抖音客服”“淘宝客服”“短信”“B站”“知乎”或“WhatsApp”。
   - 输入：`合同怎么签？`
   - 点“模拟回复”。
   - 说明：不同平台消息会先被封装成统一格式，再调用同一套知识库和客服引擎。

8. 黑卫士七维AI营销系统
   - 从 1 到 7 讲闭环：
     内容观察、内容生成、矩阵分发、评论私信分级、自动私信生成、客服知识库承接、成交跟进。
   - 重点说明合规边界：学习结构，不复制内容；私信和站外引导默认人工确认。

9. 自动私信生成器
   - 私信平台填：`抖音私信`
   - 客户称呼填：`林女士`
   - IP 省份填：`广东`
   - IP 城市填：`深圳`
   - 需求对象选：`家人`
   - 作品平台填：`抖音`
   - 作品/账号位置填：`深圳本地母婴账号`
   - 作品标题填：`深圳孩子过敏护理清单`
   - 留言内容填：`我家孩子也反复过敏，想要一份护理清单`
   - 留言和作品的关系填：`留言提到孩子反复过敏，与作品护理清单直接相关`
   - 情绪判断选：`正向 / 有兴趣`
   - 我们能提供什么填：`儿童过敏护理咨询`
   - 资料/优惠/小权益填：`免费护理清单和首次咨询优惠`
   - 点“生成一次性私信”。
   - 看结果里是否出现：可发送私信、来源追踪、发送判断、信号判断、知识库路由/知识库方向、核验清单、邀约目标、邀约决策。
   - 点“加入审批队列”，在下方“私信审批队列”里看到待审私信，再演示“通过、拒绝、已发送、归档”。
   - 说明：当前生成的是客服建议，不会自动发给客户，必须先进入私信审批队列。

10. 客户生命周期
    - 展示客户从陌生、已互动、已咨询、高意向、待成交到已成交的流转方式。
    - 说明：当前 1.0 用演示数据说明流程，真实平台线索待凭证配置后接入。

11. 私信/评论转化剧本
    - 展示资料领取、价格咨询、案例咨询、活动报名、转人工等剧本。
    - 说明：剧本是客服跟进建议，不直接替客服批量发送。

12. Hermes 指令收件箱 / 双向中心
    - 先看顶部三块统计：用户发给 Codex 的指令、Codex 回传给用户的阻塞/进度、高优先级。
    - 在表单里演示方向、类型、目标、模块和任务字段：`direction`、`type`、`target`、`moduleId`、`taskId`。
    - 说明：用户发来的任务先进入收件箱，人工确认后再处理；Codex 遇到阻塞或需要资料时，通过同一个中心回传中文说明。

13. 并行任务调度
    - 展示工作流、子任务、合并关口和 Hermes 双向规则。
    - 说明：该面板读取 `/api/orchestration-plan`，平台并行线会显示 20 个端口的任务池，企业微信和抖音优先推进，其他平台先预留。
    - 点“回传阻塞/进度”可以演示前端调用 `POST /api/hermes/blocker-report`；真实回传内容由后端接口落库。

14. 高意向线索
    - 展示刚生成的线索等级、分数、信号和建议回复。
    - 说明：后续会接到留言、私信和群内互动，形成统一线索池。

## 4. Demo 成功标准

- 后台能打开。
- `npm run demo:check` 通过。
- `npm test` 通过。
- 平台凭证配置中心能显示各平台缺少的配置，并且密钥不回显。
- 知识库能新增、问答能生成回答。
- 端口模拟器能返回模拟回复。
- 黑卫士七维AI营销系统看板能完整展示 7 大模块。
- 自动私信生成器能根据留言上下文生成一次性私信建议，并展示来源追踪、发送判断、信号判断、知识库路由/知识库方向、核验清单、邀约目标和邀约决策。
- 私信审批队列能把生成的话术放入待审，并支持通过、拒绝、已发送和归档。
- 知识库导入模板能从本机工具一键生成。
- 客户生命周期、私信/评论转化剧本和 Hermes 指令收件箱 / 双向中心能在后台看到。
- 并行任务调度能展示 `/api/orchestration-plan` 的计划；接口未就绪时要显示等待状态，而不是影响整页加载。
