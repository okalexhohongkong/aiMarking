# 黑卫士七维AI营销系统配置清单

这份清单用于准备 1.0 Demo 和后续真实平台接入。当前版本可以先演示后台、知识库、知识库导入模板、端口模拟器、自动私信生成器、私信审批队列、客户生命周期、私信/评论转化剧本、Hermes 指令收件箱 / 双向中心和并行任务调度。企业微信、抖音、个人微信、内容平台和电商平台的真实收发，需要等账号权限、平台审核和凭证配置完成后再逐个打开。

## 1. 企业微信后台准备

请管理员进入企业微信管理后台，创建“智能机器人”，拿到：

- `WECOM_BOT_ID`
- `WECOM_BOT_SECRET`
- `WECOM_WS_URL`，只有私有部署企业才需要

普通“群机器人 Webhook”不适合这个需求，因为它主要用于往群里推送消息，不能像客服一样接收群里 @ 它的问题。

## 2. 填写本机配置

推荐先在后台打开“平台凭证配置中心”，按平台填写企业微信、抖音、个人微信、内容平台、电商、短信和海外社媒的配置项。系统只显示“已配置/未配置”，不会把 `Secret`、`Token`、`API Key` 等密钥明文回显到页面。

如果用源码方式运行，也可以手工打开本目录的 `.env` 填写。桌面安装版会把配置保存在本机应用数据目录。

打开本目录的 `.env`，填写：

```bash
WECOM_BOT_ID=企业微信后台给的机器人ID
WECOM_BOT_SECRET=企业微信后台给的机器人Secret
OPENAI_API_KEY=CCX或OpenAI兼容接口访问密钥
```

如果你的行业知识库有专门模型名，把它写到：

```bash
OPENAI_MODEL=你的知识库模型名
```

如果它需要特别的答题规则，把规则写到：

```bash
KNOWLEDGE_SYSTEM_PROMPT=只根据本地知识库回答，无法确认时转人工。
```

## 3. 检查配置

后台“平台凭证配置中心”会直接显示每个平台缺什么。保存配置后，建议重启后台或桌面 App，让真实平台接入服务重新读取配置。

```bash
npm run doctor
```

看到“基础配置已填写”和“知识库接口连通性：正常”后，再启动：

只用后台维护知识库：

```bash
npm run admin
```

完整启动后台和企业微信机器人：

```bash
npm start
```

后台地址：

```text
http://127.0.0.1:8787
```

## 4. 放进群

把企业微信智能机器人加入需要服务的群。群成员这样问：

```text
@黑卫士 合同怎么签？
```

机器人只会在被 @ 时回答，不会读取普通闲聊来触发知识库。

## 5. 知识库和知识图谱怎么填

每条知识建议包含：

- `标题`：这条知识的主题
- `分类`：售前、售后、技术支持、合同、财务等
- `标签`：方便搜索的关键词
- `适用场景`：客户什么时候会问
- `关键概念`：答案依赖的对象，比如营业执照、订单号、发票
- `处理步骤`：按顺序写，一行一步
- `内容`：完整标准答案

系统会自动生成知识图谱关系，例如：

```text
开户资料清单 -> 属于 -> 售前
开户资料清单 -> 适用 -> 新客户开户
开户资料清单 -> 涉及 -> 营业执照
开户资料清单 -> 步骤 -> 准备营业执照
```

AI 回答时会按“结论 / 处理步骤 / 补充说明 / 参考来源”输出。

## 6. 自动私信生成器怎么用

后台里的“自动私信生成器”用于演示一次性私信承接建议。输入私信平台、客户称呼、IP 省市、作品平台、作品/账号位置、作品标题、发布时间、留言时间、留言内容、留言和作品关系、情绪判断、需求对象、解决方案、资料权益、官方核验链接、加群链接和联系入口后，系统会：

- 生成一段客服可参考的一次性私信正文。
- 拆出身份说明、联系原因、来源追踪、上下文摘要、发送判断、信号判断、知识库路由、知识库方向、需求判断、正规核验、核验清单、价值钩子、资料权益、邀约目标、邀约决策、主要下一步、备用下一步和风险提醒。
- 自动结合留言、作品/账号位置、解决方案和权益，从本地知识库里匹配相关知识方向。
- 根据来源追踪、发送判断和信号判断决定是否适合邀请进对应平台群；负面或电商平台场景会先走澄清、平台内承接或人工确认。
- 正向留言会给资料、优惠或加群/联系入口。
- 负面留言会先安抚、澄清和复核，不硬推成交。
- 自动过滤不可信链接和疑似密钥，避免把敏感信息带入话术。

1.0 版本只生成建议话术，不会自动发送给客户。生成后可以加入私信审批队列，人工标记通过、拒绝、已发送或归档；客服确认合适后再复制发送。

## 7. 知识库导入模板

后台“本机工具”里可以点“模板”，生成一份 `knowledge-import-template-时间.json`。模板会放在 `exports/` 目录，里面包含示例标题、分类、标签、适用场景、关键概念、处理步骤和正文。后续把真实行业资料按模板整理后，可以用本机导入工具进入知识库。

## 8. 客户生命周期和转化剧本

1.0 Demo 会把客户分成这些阶段，方便讲清楚后续运营动作：

- 陌生：刚看到内容，还没有明确咨询。
- 已互动：点过赞、评论、私信或领取过资料。
- 已咨询：问过价格、案例、合同、交付等问题。
- 高意向：多次互动，出现报价、合作、到店、到院、下单等信号。
- 待成交：需要人工顾问继续跟进。
- 已成交：进入交付或售后承接。
- 复购维护：适合二次触达、老客活动和转介绍。

“私信/评论转化剧本”用于把常见场景整理成固定流程，例如资料领取、价格咨询、案例咨询、活动报名、差评安抚和转人工。剧本只给客服建议，不直接替客服发送。

## 10. 端口封装方式

1.0 先把所有平台都封装成同一种消息格式：

```text
平台原始消息 -> 平台适配器 -> 统一消息 -> 知识库回答 -> 合规策略 -> 回复出口
```

后台的“端口模拟器”可以先测试这个流程。选择企业微信、抖音、淘宝、拼多多、京东、短信、头条号、百家号、B站、知乎、LinkedIn、Facebook、WhatsApp、X/Twitter、TikTok 等端口，输入一条客户消息，系统会用同一套知识库和知识图谱生成回复。以后拿到平台 API 权限后，只需要把真实消息接到对应适配器里，不需要重做知识库、话术库和线索规则。

## 11. 黑卫士七维AI营销系统

系统新增“黑卫士七维AI营销系统”总蓝图：

1. 内容观察：收集公开高表现内容，提取选题、结构、卖点、亮点和平台信号，只做原创参考。
2. 内容生成：结合自己的产品和服务，生成原创视频脚本、图文海报、标题库和故事背景库。
3. 矩阵分发：把审核通过的内容安排到不同平台和矩阵账号，真实发布待平台权限配置。
4. 评论私信分级：收集评论、留言、私信、留资，按意向、情绪、地域、时间和类型分组。
5. 自动私信生成器：对有明确互动的人生成来源透明、身份清楚、可核验、带资料权益和下一步入口的一次性私信，默认人工确认。
6. 客服知识库承接：把进群或进入客服窗口的人，按行业知识库和知识图谱持续承接。
7. 成交跟进：按客户生命周期引导到商城付费、定金、到院、到店或人工顾问成交。

配套模块：

- 客户生命周期：判断客户现在处于陌生、已互动、已咨询、高意向、待成交、已成交还是复购维护。
- 私信/评论转化剧本：把评论、私信、资料领取、报价咨询、案例咨询等场景整理成跟进建议。
- Hermes 指令收件箱 / 双向中心：同时展示“用户发给 Codex 的指令”和“Codex 回传给用户的阻塞/进度指令”。新增字段包括 `direction`、`type`、`target`、`moduleId`、`taskId`，用于标记方向、类型、目标、模块和任务。
- 并行任务调度：展示 `/api/orchestration-plan` 返回的工作流、子任务、合并关口和 Hermes 回传策略；前端只负责展示和发起阻塞/进度回传，不直接执行后端任务。

注意：系统会把“学习头部内容”处理成原创结构参考，不做逐句仿写或素材搬运；批量私信、站外引流、付款引导、医疗金融等关键承诺，默认需要人工审核。

## 12. Hermes 双向中心和并行任务调度

新版后台把原来的 Hermes 指令收件箱升级成双向中心：

- 入站方向：用户通过 Hermes 给 Codex 下发任务，先进入收件箱，人工接受后再执行。
- 出站方向：Codex 遇到阻塞、资料缺口或需要同步进度时，通过 Hermes 回传中文说明。
- 指令类型：`task` 表示用户指令，`blocker` 表示阻塞说明，`progress` 表示进度同步，`split` 表示任务拆分，`merge` 表示合并关口。
- 展示字段：`target` 看这条指令发给谁，`moduleId` 看属于哪个模块，`taskId` 看对应哪一个任务。

并行任务调度面板会读取：

```text
GET /api/orchestration-plan
```

阻塞/进度回传按钮会调用：

```text
POST /api/hermes/blocker-report
```

当前前端已经按这些接口和字段展示。后端在保存 Hermes 命令时也需要保留 `direction`、`type`、`target`、`moduleId`、`taskId`，这样刷新后才能完整看到双向指令元数据。
