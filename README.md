# 黑卫士七维AI营销系统

黑卫士七维AI营销系统是一个本机可演示的 1.0 版本，用来展示从客户咨询、内容互动、私信承接到人工确认跟进的完整流程。当前已经可以打开后台、维护知识库、测试问答、查看多平台端口、模拟客户消息，并演示自动私信生成器、客户生命周期、私信/评论转化剧本和 Hermes 指令收件箱等模块。

当前版本不夸大为“已经接通所有真实平台”。企业微信、抖音、小红书、快手、视频号、电商平台和个人微信等入口已经预留或封装，真实收发需要等管理员配置平台权限、账号凭证和审核条件后，再逐个接入。

## 当前已经做好的部分

- 管理后台：本机浏览器可打开。
- 标题进度徽章：后台每个主要模块标题后显示倒计时和进度百分比；蓝色表示正常推进，红色表示暂停或被凭证、权限等外部条件卡住，绿色表示超前或提前完成。
- 4 小时进度汇报规则：正常冲刺期间，每 4 小时同步一次“已完成、正在做、剩余倒计时、是否卡住”。
- 本机加急工具：一键启动、停止、状态检查、数据备份、数据恢复、知识库导入导出。
- 平台凭证配置中心：后台集中填写企业微信、抖音、个人微信、内容平台、电商、短信和海外社媒接入资料；页面只显示已配置/未配置，不回显密钥明文。
- 知识库增删改查，适合维护售前、售后、合同、交付、活动和转人工说明。
- 知识图谱：每条知识可维护分类、适用场景、关键概念、处理步骤，并自动形成关系图。
- 结构化回答：系统按“结论 / 处理步骤 / 补充说明 / 参考来源”组织答案。
- 多平台客服端口：企业微信、小程序、个人微信、抖音、快手、小红书、视频号、淘宝、拼多多、京东、短信、头条号、百家号、B站、知乎、LinkedIn、Facebook、WhatsApp、X/Twitter、TikTok 等入口已经预留。
- 端口模拟器：可以先模拟平台消息，验证同一套知识库、合规策略和人工确认流程。
- 黑卫士七维AI营销系统看板：把内容观察、内容生成、矩阵分发、评论私信分级、自动私信生成、客服承接和成交跟进串成一条可讲清楚的演示闭环。
- 自动私信生成器：根据平台、IP 地域、作品/账号位置、发布时间、留言、情绪、需求对象、知识库方向、正规核验入口、资料权益和联系入口生成一次性私信建议，并显示来源追踪、发送判断、信号判断、知识库路由、核验清单、邀约目标和邀约决策，生成后可加入私信审批队列，默认先人工确认再发送。
- 知识库导入模板：后台“本机工具”可一键生成知识库导入模板，方便把真实行业资料整理成可导入格式。
- 客户生命周期：按陌生、已互动、已咨询、高意向、待成交、已成交、复购维护等阶段管理线索。
- 私信/评论转化剧本：把评论、私信、资料领取、报价、案例、活动咨询等场景整理成可复用跟进剧本。
- Hermes 指令收件箱：预留接收人工指令、平台任务和后续自动化命令的位置，1.0 先做演示和记录，不直接替人批量发送。
- 首批种子内容：内置售前、售后、合同、资料领取、电商合规、转人工等基础知识和话术。
- 问答测试和最近消息日志，用于演示咨询承接和人工复盘。

## 多平台端口说明

系统后台会显示每个端口的接入状态、需要填写的配置项、能力范围和合规策略。1.0 已经把端口层规划好，后续拿到平台权限后逐个接适配器，不需要重做知识库、话术库和线索规则。

后台的“平台凭证配置中心”可以直接填写这些端口的本机配置。为了安全，页面和接口不会把密钥明文显示出来，只显示“已配置/未配置”。保存后建议重启后台或桌面 App，让真实平台接入服务重新读取配置。桌面安装版会把配置保存在本机应用数据目录；源码运行版默认保存在项目 `.env`。

当前接入路线图：

```text
第一优先级：企业微信，先做真实机器人收发验证。
第二优先级：抖音，承接客服、私信和线索互动。
第三优先级：个人微信，通过 EasyClaw 或本地信使中转，默认人工确认。
后续批次：小红书、快手、视频号、电商客服、短信、头条号、百家号、B站、知乎、领英和海外社媒端口。
```

电商客服端口默认只做平台内承接，不直接生成“加个人微信、加企微、转私域”的站外引导话术。内容平台和个人微信端口也默认走人工确认，避免账号风控。

现在后台提供“端口模拟器”。可以先选企业微信、抖音、淘宝、拼多多、京东、短信、B站、知乎、WhatsApp 等端口，输入一条客户消息，看它如何进入统一消息流程，再交给同一套知识库和回复引擎处理。等后面接真平台时，只换最外层收发适配，不动中间知识和规则。

## 企业微信真实接入要准备什么

1. 管理员登录企业微信管理后台，创建或启用“智能机器人”。
2. 拿到机器人身份和密钥，填进本机配置。
3. 把机器人加入需要服务的群，或通过企业微信后台配置可见范围。
4. 注意普通“群机器人 Webhook”主要用于主动推送，不适合接收群里 @ 它的问题；本项目预留的是智能机器人长连接能力。

1.0 Demo 可以不填这些真实凭证；缺凭证时只演示后台、模拟端口和本机知识库。

## 知识库怎么填

录入知识时建议按这个结构准备内容：

```text
标题：这条知识解决什么问题
分类：售前 / 售后 / 技术支持 / 财务 / 合同
标签：用于搜索的关键词
适用场景：客户在什么情况下会问
关键概念：答案里涉及的核心对象
处理步骤：一行一步，形成可执行流程
内容：标准答案、注意事项、例外情况
```

回答时系统会优先使用“适用场景、关键概念、处理步骤”组织逻辑，不只是简单搜索正文。

## 黑卫士七维AI营销系统

后台的黑卫士七维AI营销系统看板，用来演示七个维度：

1. 内容观察：整理公开高表现内容的选题、结构、情绪钩子和平台信号，只做原创参考。
2. 内容生成：结合自己的产品卖点，生成视频脚本、图文标题、活动说明和素材方向。
3. 矩阵分发：把审核通过的内容按平台、账号和时间安排分发计划，真实发布待平台权限配置。
4. 评论私信分级：把评论、私信、留资按意向、情绪、地域、时间和类型分组。
5. 自动私信生成器：为有明确互动的人生成来源透明、身份清楚、可核验、带资料权益和下一步入口的一次性私信，并先判断地域、作品/账号位置、留言关系、发送条件、知识库路由和邀约方式，默认人工确认。
6. 客服知识库承接：用行业知识库和知识图谱承接群、企微、平台客服里的持续对话。
7. 成交跟进：按客户生命周期给出下一步建议，引导到商城、定金、到店、到院或人工顾问。

配套模块包括：

- 客户生命周期：看客户从陌生到复购维护处在哪一步。
- 私信/评论转化剧本：把不同互动场景沉淀成可复用跟进流程。
- Hermes 指令收件箱：集中接收后续人工指令、平台任务和待处理动作，1.0 先预留入口和演示口径。

合规边界会长期保留在系统里：学习爆款结构但不复制内容；批量发布、批量私信、站外引导默认人工确认；电商平台优先平台内承接；医疗、金融等高风险承诺必须人工审核。

## 本机配置

首次运行可复制配置模板，再按需要填写：

```bash
cp .env.example .env
```

最少需要关注这些配置：

```text
企业微信机器人身份和密钥：真实接入企业微信群时填写。
AI 接口访问密钥：需要调用外部模型时填写。
后台端口：默认使用 8787。
```

如果只是做 1.0 Demo，可以先不填真实平台凭证，后台和端口模拟仍然可以展示。

也可以不手工编辑 `.env`，直接在后台“平台凭证配置中心”里填写。该中心会按平台分组显示缺少哪些资料，适合后续逐个接入企业微信、抖音、个人微信和其他平台。
